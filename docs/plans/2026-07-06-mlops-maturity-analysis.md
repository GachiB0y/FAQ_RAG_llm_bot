# MLOps-зрелость проекта: 7 болей и как их лечить

**Дата:** 2026-07-06
**Контекст:** прикладываем классический список «7 болей ML без MLOps» (из курса) к
нашему RAG-проекту. Честно, где мы виноваты, и какими актуальными инструментами
2026 это лечится.

> Методология та же что в clean-experiment: не приукрашиваем. Проект — учебный,
> и по MLOps-меркам мы пока на уровне «ноль-один». Это нормально для pet-проекта,
> но полезно видеть разрыв до прода.

---

## Диагностика: где мы по каждой из 7 болей

| # | Боль | Наш статус | Оценка |
|---|---|---|---|
| 1 | **Много ручного труда** | Каждый eval-прогон запускали руками через `docker exec`, следили за прогрессом через мониторы, ловили 429-ки вручную. Контекст терялся между сессиями (перезапуски) | 🔴 сильно болит |
| 2 | **Ничего не версионируется** | Код — в git ✓. Но: `kg.json`/`testset` закоммичены сырыми (без версий/хэшей), `mlruns/` в .gitignore, нет привязки «эта модель + эти данные + этот промпт = этот результат». Промпты в коде, версий нет | 🔴 болит |
| 3 | **Разрыв между ML и Ops** | Docker есть ✓, но всё поднимали руками. Нет IaC, нет CI/CD, деплой = «docker compose up на моём маке» | 🟡 частично |
| 4 | **Разрыв в мониторинге** | **Полностью отсутствует.** Нет мониторинга ни инфры (кроме `docker stats` глазами), ни качества модели в проде. Если RAG начнёт галлюцинировать — не узнаем | 🔴 сильно болит |
| 5 | **Локальные эксперименты, бардак** | Классика: Ollama OOM-ила, конфликты зависимостей (ragas vs langchain), OpenRouter лимиты. Окружение — «работает на моей машине». Нет lockfile, нет devcontainer | 🔴 болит |
| 6 | **Разрыв между ML и ML** | Мы соло → коммуникации нет. Но результаты экспериментов жили в моей голове + чате, а не в общем трекере. MLflow частично это закрыл | 🟢 слабо (соло) |
| 7 | **Разрыв в тестировании** | `backend/tests/` есть (pytest для loader/auth), eval-скрипты отдельно. Модель (Ragas) и инфра (pytest) тестируются раздельно, **нет CI** который гоняет и то и другое | 🟡 частично |

**Итого:** сильнее всего болят **#1 (ручной труд), #2 (версионирование), #4 (мониторинг), #5 (окружение)**.

## Чем лечить — актуальный MLOps-стек 2026 под каждую боль

### Боль #1 — Ручной труд → оркестрация + one-command pipeline

| Инструмент | Что даёт |
|---|---|
| **Makefile / Taskfile** | `make eval` вместо 5 строк `docker exec -e ... python ...`. Быстрый первый шаг |
| **Prefect / Dagster** | Пайплайн ingest→kg→testset→eval как flow с retry, расписанием, UI |
| **CLI (Typer/Click)** | Обернуть скрипты в `python -m rageval run --hybrid --judge=...` |

### Боль #2 — Версионирование → DVC + Model/Prompt Registry

| Инструмент | Что версионирует |
|---|---|
| **DVC** (Data Version Control) | Данные, эмбеддинги, kg.json, testset — версии привязаны к git-коммиту |
| **MLflow Model Registry** | Мы уже используем MLflow tracking — добавить registry для конфигов RAG |
| **MLflow Prompt Registry** / **Langfuse Prompts** | Версии промптов (у нас SYSTEM_PROMPT в коде — надо вынести и версионировать) |
| **LakeFS** | Git-подобное версионирование для больших данных (если корпус вырастет) |

### Боль #3 — ML/Ops разрыв → контейнеры + IaC + CI/CD

| Инструмент | Что |
|---|---|
| **Docker** ✓ (есть) | Уже контейнеризовано |
| **GitHub Actions / GitLab CI** | Автосборка, тесты, деплой |
| **Terraform / Ansible** | IaC — инфра как код (для облака) |
| **Kubernetes + Helm** | Для масштаба (нам пока избыточно) |

### Боль #4 — Мониторинг → Langfuse + Prometheus/Grafana + drift

| Инструмент | Что мониторит |
|---|---|
| **Langfuse** | Каждый LLM-запрос: latency, стоимость, качество (online eval). **Для нашего RAG — приоритет** |
| **Prometheus + Grafana** | Инфра-метрики (CPU, память, RPS, ошибки) |
| **Evidently AI** | Data drift / embedding drift — когда вопросы юзеров «уплыли» от того что в корпусе |
| **Ragas в проде** | Online-eval на семпле реальных запросов |

### Боль #5 — Бардак в окружении → lockfiles + devcontainer

| Инструмент | Что |
|---|---|
| **uv** (замена pip/poetry) | Тренд 2025-2026: очень быстрый, lock-файл, воспроизводимое окружение. **Прямое лекарство от наших конфликтов ragas/langchain** |
| **Dev Containers** | `.devcontainer/` — VS Code поднимает идентичное окружение у всех |
| **Docker multi-stage** ✓ | Уже есть базово |

### Боль #6 — ML/ML разрыв → общий трекер (для команды)

| Инструмент | Что |
|---|---|
| **MLflow / Langfuse (shared)** | Общий сервер, а не SQLite на маке — все видят эксперименты |
| **Model cards / docs** | Документация моделей (у нас есть отчёты — это уже хорошо) |

### Боль #7 — Разрыв в тестировании → CI с eval + infra тестами вместе

| Инструмент | Что |
|---|---|
| **pytest** ✓ (есть для loader/auth) | Инфра-тесты |
| **Ragas в CI** | Прогон eval на mini-testset при каждом PR, **fail если метрики просели** |
| **GitHub Actions matrix** | Прогнать и pytest, и eval в одном workflow |

## Предлагаемый план «пощупать MLOps» (по нарастающей сложности)

### Шаг A — `uv` + lockfile (лечит #5, ~1 час)
Заменить хаотичный `pip install` на `uv` с зафиксированным lock-файлом. Больше
никаких «ragas 0.4 сломал langchain». Воспроизводимое окружение.

### Шаг B — Makefile (лечит #1, ~1 час)
`make ingest`, `make kg`, `make testset`, `make eval-dense`, `make eval-hybrid`,
`make mlflow-ui`. Вместо простыней `docker exec -e ...`.

### Шаг C — Langfuse (лечит #4, ~2-3 часа) ⭐
Self-hosted трейсинг. Оборачиваем `RAGEngine.query()` — видим каждый запрос,
latency, стоимость. Это то, чего у нас ВООБЩЕ нет.

### Шаг D — DVC (лечит #2, ~2 часа)
Версионируем `kg.json`, `testset_auto.json`, eval-результаты. Каждый git-коммит
→ известно какие данные + модель + промпт дали эти метрики.

### Шаг E — CI с Ragas (лечит #7 + #3, ~3 часа)
GitHub Actions: при PR гоняет pytest + mini eval-прогон. Если faithfulness
просел > порога — PR красный. Это «промышленный» уровень.

## Рекомендация: с чего начать прямо сейчас

**Шаг A (uv) + Шаг B (Makefile)** за один заход — быстро, лечат сразу 2 боли
(#1 и #5), и делают все дальнейшие шаги удобнее (одна команда вместо простыней).

Потом **Langfuse** (боль #4 — самая зияющая дыра) как в production-roadmap.

Это превратит проект из «набор скриптов которые я запускаю руками и помню как»
в «воспроизводимый pipeline с мониторингом» — то есть закроет ключевые боли слайда.
