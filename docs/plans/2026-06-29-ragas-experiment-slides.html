Зачем оценивать RAG автоматически?

  • Любое изменение в RAG-pipeline (модель, чанкер, retriever, prompt) — может улучшить или ухудшить качество
  • Ручная проверка не масштабируется: 5 примеров — не показатель, 100 — никто читать не будет
  • Нужен воспроизводимый, числовой способ:
    • сравнить «было vs стало»
    • блокировать регрессии в CI
    • объяснить команде «почему это лучше»

Цель и гипотеза

Цель

  1. Освоить Ragas + MLflow на реальном проекте
  2. Измерить качество текущего RAG
  3. Проверить: даёт ли hybrid search прирост на нашем корпусе?

Гипотеза

На корпусе с формальными ссылками (ФЗ № 7-ФЗ, § 21.3, БЕКОСО, МКПС) dense-vector search проигрывает sparse-поиску по recall:

  • Embedding-модель «понимает темы», но плохо различает близкие термины
  • BM25 ищет по лексическому совпадению → находит точные ссылки моментально
  • RRF-фьюжен даёт лучшее из двух миров

Архитектура pipeline

PDF → chunks → Knowledge Graph → Test set (с personas) → RAG → Ragas → MLflow
       (512т)   (Ragas)            (20 вопросов)             (top_k=5)  judge   (compare)

Метрики Ragas (LLM-as-judge)

Про LLM-генератор

  • Faithfulness
    Нет ли галлюцинаций? Доля claims в ответе, подкреплённых контекстом
  • Answer Relevancy
    Отвечает ли на вопрос? Cosine между «восстановленным» вопросом и оригиналом

Про retrieval

  • Context Precision (with ref)
    Не приволок ли мусор? Доля релевантных чанков в top-k
  • Context Recall
    Нашёл ли всё нужное? Доля claims из reference, выводимых из контекста

Что произошло на каждой метрике

Context Recall: 0.736 → 1.000 (+36%)

Dense пропускал ~25% нужной инфы — типично на вопросах со ссылками на статьи и ФЗ. BM25 находит точные совпадения, RRF поднимает их в топ-5.

Context Precision: 0.606 → 0.917 (+51%)

При dense-only 40% top-5 были мусором → LLM отвлекалась. Hybrid отсёк нерелевантные.

Faithfulness: 0.826 → 1.000 (+21%)

Когда контекст полный, LLM не нужно «достраивать» из общих знаний → нет галлюцинаций. Идеал 1.0.

Answer Relevancy: 0.700 → 0.816 (+17%)

Точнее контекст → точнее ответ по сути. Особенно заметно на multi-hop.

Что это означает в практике

  • На корпусе с формализмами (ФЗ, статьи, аббревиатуры) hybrid обязателен
  • Embedding-модели «понимают темы», но проваливаются на точных терминах
    (ФЗ № 115 vs ФЗ № 150 — практически одинаковый embedding)
  • BM25 эти случаи решает из коробки, добавление ~20 строк кода
  • Самое дешёвое улучшение RAG в данном кейсе

Что попадёт в прод

  • enable_hybrid=True в QdrantVectorStore
  • Переиндексация существующих 418 чанков с sparse-векторами
  • BM25 как часть штатного pipeline

Что мы узнали об инструментах

Ragas

  • Knowledge graph + Personas → реалистичный тестовый набор, не «придумай 10 вопросов»
  • 4 метрики покрывают и LLM-генератор, и retrieval независимо
  • LLM-as-judge — настолько умное, насколько умна модель-судья

MLflow

  • Compare-режим для двух run-ов — самое ценное
  • Логирование by_synth__* срезов даёт детализацию по типу вопроса
  • SQLite-backend + локальный артефакт-стор → подходит для self-hosted

Ограничения эксперимента

Методологические

  • 20 вопросов — статистика хлипкая. Стоит расширить до 50–100
  • gemma-4-31b был и testset-генератором, и судьёй → потенциальный bias.
    Для прода нужен Claude Haiku / GPT-4 в роли судьи
  • Картинка-PDF парсится в кракозябры — BM25 цепляет мусор. Нужен OCR

Технические

  • OpenRouter free имеет лимиты (20 RPM, 2000/день), регулярные 429-ки
  • Ollama на Mac OOM-ит при нескольких моделях — настройка num_ctx

Дальнейшие шаги

Короткие (1–2 дня)

  1. Hybrid в прод: retriever.py → enable_hybrid=True + реиндексация
  2. Платный судья: Claude Haiku 4.5 ($0.10/прогон) → стабильные оценки
  3. OCR: pytesseract для картинок-PDF

Средние (1–2 недели)

  1. Расширение датасета до 50–100 вопросов + ручная курация
  2. CI-интеграция: GitHub Action, падает если faithfulness просел > 5%

Длинные (1–2 месяца)

  1. Prefect для оркестрации ingest и nightly eval
  2. Полный корпус ФПСР (10 директорий с Яндекс-Диска)
  3. Реальные user-вопросы вместо/в дополнение к auto-generated
