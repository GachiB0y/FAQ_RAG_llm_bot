# Каталог правил и навыков Cursor

> Все файлы правил лежат в `.cursor/rules` и подчиняются иерархии FSD. Навыки (skills) находятся в `.cursor/skills` и описывают переиспользуемые "рецепты" действий. Этот каталог поможет быстро найти нужное правило или навык, понять сферу действия и ссылку на исходник.

## Правила vs Навыки

- **Правила (rules)** — описывают **что** должно быть в проекте: архитектура, код-стайл, требования к UI, data layer и т.д. Применяются автоматически при работе с проектом.
- **Навыки (skills)** — описывают **как** создать что-то: настройка FSD-проекта, создание фичи, настройка React Query и т.д. Вызываются явно через упоминание или контекст задачи.

> 📚 Полный каталог навыков: `.cursor/skills/skills-index.mdc`

## 1. Структура папок

```
.cursor/
└── rules/
    ├── core/          ← фундаментальные правила (архитектура, стиль)
    ├── stack/         ← договорённости по стеку (UI, i18n, data, routing, theme)
    ├── assets/        ← работа с графикой и ресурсами
    ├── agent/         ← как взаимодействовать с агентом и передавать требования
    └── tooling/       ← инструменты, линт, процессы
```

> Для фич и страниц создаём локальный каталог `.cursor/rules` прямо внутри слоя (`src/features/foo/.cursor/rules/foo.mdc`), где описываем частные требования.

## 2. Мастер-индекс

- Главный навигатор: `.cursor/rules/rules-index.mdc` — маркерованный список с краткими тезисами и ссылками на глобальные правила.

## 4. Как добавлять новые правила

1. Создай папку для категории (`core`, `stack`, `assets`, `agent`, `tooling`) либо локальную в слое.
2. Положи `.mdc` с метаданными (`name`, `description`, `globs`).
3. Обнови таблицу выше, чтобы команда видела новое правило.
4. Если правило меняет процесс/стек, добавь ссылку в `docs/instructions.md` или README соответствующего слоя.

## 4. Навыки (Skills)

Навыки находятся в `.cursor/skills/` и организованы по категориям:

- **Setup** — настройка проекта (FSD, React Query, i18n, Git workflow)
- **Create** — создание компонентов/фич (feature, table, switcher, page)
- **Configure** — конфигурация инструментов (создание правил)

Полный список: `.cursor/skills/skills-index.mdc`

### Примеры использования навыков

- "Создай фичу по FSD" → применяется навык `create-feature`
- "Настрой React Query" → применяется навык `setup-react-query-axios`
- "Создай таблицу с пагинацией" → применяется навык `create-table-with-pagination`
